3. 認証・ユーザー管理

3.1 認証フロー

認証方式: メール OTP（パスワードレス）
  1. ユーザーがメールアドレスを入力
  2. Supabase Auth がワンタイムパスワード（6桁）をメール送信
  3. ユーザーがOTPを入力 → セッション確立
  4. 初回ログイン時に CareerPath ID を紐づけ（任意）

パスワードレスを選択する理由: 外国人留学生はパスワード管理に不慣れなケースが多い。OTPなら「メールを開いて数字を入力するだけ」でシンプル。

3.2 ユーザーロール

ロール権限対象者
student自分のデータの読み書き留学生（一般ユーザー）
admin全ユーザーの進捗閲覧, コンテンツ管理ヒトキワスタッフ（Admin等）
school_admin所属校の学生の進捗閲覧学校の担当者（将来拡張）

CREATE TABLE courses (

  title       JSONB NOT NULL,          -- {"en":"...","ja":"...","vi":"..."}

CREATE TABLE lessons (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),

4.3 Row Level Security (RLS)

-- profiles: 自分のプロフィールのみ編集可。adminは全員閲覧可
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users read own profile" ON profiles
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users update own profile" ON profiles
  FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Admin reads all profiles" ON profiles
  FOR SELECT USING (
    EXISTS(SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );

-- lesson_progress: 自分の進捗のみ。adminは全閲覧
ALTER TABLE lesson_progress ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users manage own progress" ON lesson_progress
  FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Admin reads all progress" ON lesson_progress
  FOR SELECT USING (
    EXISTS(SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );

-- 同様のパターンを qa_views, quiz_results にも適用

6.3 Supabase Edge Function: 管理者統計

// supabase/functions/admin-stats/index.ts
import { createClient } from '@supabase/supabase-js'

Deno.serve(async (req) => {
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  // KPIクエリ
  const [users, active, progress] = await Promise.all([
    supabase.from('profiles').select('*', { count: 'exact' }),
    supabase.from('profiles')
      .select('*', { count: 'exact' })
      .gte('last_active_at', sevenDaysAgo()),
    supabase.rpc('avg_completion_rate')
  ])

  return new Response(JSON.stringify({
    total_users: users.count,
    active_7d: active.count,
    avg_progress: progress.data,
  }))
})

6.4 レッスンロック機能（順序強制）

設計方針: 「やらない人が出るので、強制させて全部完了で自由を与えたい」
  • レッスンロック: 同一コース内で、前のレッスン（sort_order が小さい方）が completed でなければ次のレッスンにアクセス不可
  • コース順序強制: コース1（身だしなみ）の全レッスン完了 → コース2（履歴書）解放、のように段階的アンロック
  • 自由モード: 全4コースのレッスンをすべて完了すると、ロック解除。どのレッスンにも自由にアクセス可能に
  • 管理画面: 「誰がどこで止まっているか」を一覧表示
  • アカウント発行: 管理者がアカウント作成 → 学生にURL+IDを共有（セルフサインアップなし）

ロック判定ロジック（フロントエンド）

// レッスンがアクセス可能かを判定する関数
function isLessonUnlocked(lessonId, courseId, userProgress) {
  // 全コース完了者は自由モード
  if (userProgress.allCoursesCompleted) return true;

  // 前のコースが未完了ならロック
  const course = courses.find(c => c.id === courseId);
  const prevCourses = courses.filter(c => c.sort_order < course.sort_order);
  for (const pc of prevCourses) {
    if (!isCourseCompleted(pc.id, userProgress)) return false;
  }

  // 同一コース内: 前のレッスンが完了しているか
  const lesson = lessons.find(l => l.id === lessonId);
  const prevLessons = lessons.filter(
    l => l.course_id === courseId && l.sort_order < lesson.sort_order
  );
  return prevLessons.every(pl =>
    userProgress.lessons[pl.id]?.status === 'completed'
  );
}

9. PWA・オフライン対応

なぜPWAか
  • 留学生は App Store のアカウント問題（国の切り替え等）を抱えがち
  • URLをLINEやMessengerで共有するだけでアクセス可能
  • Apple 30%手数料が不要（課金時）
  • 電車内（地下）でも学習可能なオフライン機能

キャッシュ戦略

リソース戦略理由
HTMLシェル・CSS・JSCache First即座に画面表示
レッスンコンテンツStale While Revalidateオフラインでも読める + 最新化
Q&AデータStale While Revalidate同上
進捗データNetwork First最新の進捗を優先
フォント・画像Cache First (30日)変更頻度が低い
